ATT（App Tracking Transparency）

iOS 14.5+ 必須のトラッキング許可ダイアログ制御仕様

v1.0 機能ID: RS-FEAT-ATT

概要

Apple の App Tracking Transparency（ATT）フレームワークに基づき、AdMob による IDFA（広告識別子）利用前にユーザーの許可を求める。 許可状況に応じて GoogleMobileAds v12 がパーソナライズ広告と非パーソナライズ広告を自動で切り替える。

関連画面

  • ホーム（認証後の初回表示時にプロンプト要求）
  • 設定（システム設定への誘導リンク）

プロンプト表示タイミング

ContentView の authenticated 遷移（ホーム表示後）で ATTrackingManager.requestTrackingAuthorization を1回要求する。
条件動作
アプリが active のときプロンプトを表示する
アプリが inactive のときプロンプトを表示しない（OSが無視するため）
scenePhase が .active に復帰したとき未許諾（.notDetermined）であれば再試行して取り逃しを防ぐ
すでに許可 / 拒否済み再プロンプトは表示しない（OS 制御）

処理フロー

sequenceDiagram participant App as ContentView participant ATT as ATTrackingManager participant GAD as GoogleMobileAds App->>App: authenticated 遷移 (ホーム表示) App->>App: scenePhase == .active を確認 App->>ATT: requestTrackingAuthorization() ATT-->>App: status (.authorized / .denied / .restricted / .notDetermined) alt status == .authorized App->>GAD: IDFA 利用 (パーソナライズ広告) else status != .authorized App->>GAD: IDFA 非利用 (非パーソナライズ広告) end Note over App: scenePhase 復帰時に .notDetermined なら再試行

NSUserTrackingUsageDescription

Info.plist に設定するユーザー向け説明文（App Store 審査で具体性を求められる）:
あなたの興味・関心に合った広告を表示するため、広告識別子（IDFA）の利用許可をお願いしています。許可しない場合も、アプリの全機能をご利用いただけます。

GoogleMobileAds v12 との連携

ATT ステータスAdMob の動作
.authorizedIDFA を利用したパーソナライズ広告を配信
.denied / .restrictedIDFA を利用しない非パーソナライズ広告を配信
.notDetermined非パーソナライズ広告（プロンプト未回答の間）
GoogleMobileAds v12 は ATT の許可状況を自動で検出し、IDFA 利用 / 非パーソナライズを切り替える。 アプリ側でフラグを手動設定する必要はない。

ビジネスルール

  • ATT プロンプトはアプリ起動中に1度のみ表示（iOS OS 制御）
  • 許可・拒否にかかわらず、アプリの全機能を利用可能とする
  • 設定アプリから後から変更可能であることを設定画面で案内する
  • プロンプトは active 状態のときのみ要求し、inactive 時は要求しない

エラー処理

発生条件対応
requestTrackingAuthorization が inactive 状態で呼ばれたscenePhase == .active のガードで防止
プロンプト表示後も .notDetermined のままscenePhase 復帰時に再試行

実装メモ

  • ContentView の authenticated 分岐後に ATT 要求処理を実装
  • scenePhase 監視で復帰時の再試行を実装
  • NSUserTrackingUsageDescription を具体的な文言で Info.plist に設定
  • GoogleMobileAds v12 の自動切替を確認済み
