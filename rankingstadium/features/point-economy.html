ポイント経済

アプリ内通貨「ポイント」の獲得・消費の仕組み

v1.1 機能ID: RS-FEAT-POINT

概要

ランキング作成・再投票・閲覧解放などに「ポイント」を消費。日次ログインボーナス・広告報酬で獲得。 二重付与防止のためクライアントとDB両方で確認する二層チェック構造。

関連画面

  • HomeView / MyPageView（ポイント残高表示）
  • 結果ロック画面（解放消費）
  • ランキング作成（作成消費）
  • 投票画面（再投票消費）

無料作成枠

新規ユーザーには freeCreationRemaining として無料でランキング作成できる枠が用意される。 消費前にこのカウンタを優先利用する。

ビジネスルール

  • 消費操作は必ず PointService.consumePoints() 経由（直接 users.points を更新しない）
  • すべての取引は PointTransaction に履歴として記録
  • ログインボーナスは lastLoginDate（クライアント） + point_transactions（DB）の二層で重複防止
  • 連続ログイン日数判定は DateComponents の year/month/day を個別 Int 比較。DateComponents == 演算子は内部の calendar/timeZone も比較対象に含まれるため、参照インスタンスが異なると偽 false 判定（連続日でも非連続扱い）となるバグを根本回避
  • users.last_login_date は DATE 型カラムのため、クライアントは Date 型ではなく ローカルTZ基準の "yyyy-MM-dd" 文字列（pointDateOnlyFormatter）で送信する。Date 型送信時の UTC ずれ（JST 端末で前日扱い）を防ぐため
  • ログインボーナスは中央ポップアップで表示（OKボタンまたは半透明背景タップで閉じる）。旧仕様の上部トースト（3秒自動消去）から変更
  • 残高不足時は RankingServiceError.insufficientPoints(required:current:) で必要量を返す
  • 下書き削除時、24時間以内かつ未公開なら cancelRefund でポイント返却

実装メモ

  • PointService.consumePoints() でポイント減少 + point_transactions INSERT を実行
  • PointAction enum で15種のアクションを定義
  • claimLoginBonus() で日次付与（連続ログイン日数カウント: 1日目=1pt / 2日目=2pt / 3日目以降=3pt）
  • ログインボーナス UI: 中央ポップアップ（OKボタン or 半透明背景タップで閉じる）
  • 日付比較は DateComponents の year/month/day を 個別 Int 比較。DateComponents == の calendar/timeZone 参照差による偽 false 判定を排除
  • 広告報酬（claimAdReward）の付与は RewardedAdManager.shared.show(onReward:) による GADRewardedAd 視聴完了コールバック内のみで発火。広告を見ずに付与される loophole は構造的に存在しない
  • Supabase Edge Function 化（v1.1検討）でアトミック保証
