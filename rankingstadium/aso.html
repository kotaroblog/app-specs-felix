ASOキーワード戦略

App Store Optimization の方針とキーワード設計（初版・要レビュー）

本ページは初版です。マーケ担当・友達と協議して埋めてください。検索ボリューム・競合強度は仮置きです。

ターゲットユーザー

  • ペルソナ10〜30代、SNSやエンタメ系アプリで遊び慣れている層
  • 課題推しランキング・好きなものランキングを共有したい、人の意見を見たい
  • 利用シーン通勤・休憩・夜の暇つぶし、推し活、議論したいテーマがあるとき

App Store Connect 設定（案）

  • アプリ名（30字）[要検討] 例: ランスタ - みんなの推しランキング投票
  • サブタイトル（30字）[要検討] 例: 好きなものTOP3を投稿・投票
  • プロモテキスト（170字）[要記入]
  • キーワードフィールド（100字）ランキング,投票,推し,意見,TOP3,クイズ,暇つぶし,SNS,コミュニティ

説明文の構造（推奨）

  1. 1行目（最重要）: 「自分のランキングを作って、みんなに投票してもらおう」
  2. 2〜3行目: 推し活・好きなもの議論のシーン提示
  3. 機能リスト: ランキング作成 / TOP3投票 / クイズゲート / フォロー / 通知
  4. 使い方: 簡単3ステップで作成・公開
  5. 差別化ポイント: クイズで意識の高い投票者だけに絞れる、ポイント経済による質の確保
  6. CTA: 「今すぐ無料でダウンロード」

スクリーンショット戦略（推奨順序）

順序テーマキャッチコピー候補
1第一印象「みんなの推しを、ランキングに」
2ランキング作成「3ステップで誰でも作れる」
3投票（TOP3）「シングル / TOP3 から選べる」
4クイズゲート「クイズに正解した人だけ投票できる」
5結果画面 / シェア「結果を友達と共有しよう」

レビュー対策

  • ★4以上を維持する目標
  • レビュー依頼ダイアログのタイミング: 投票3回完了 / 起動5回 等で検討
  • 低評価レビューには丁寧返信、機能改善要望を月次で集約
  • FeedbackView から App内フィードバック導線を確保し、App Store レビューでの不満集中を回避

本番リリース前のチェック

  • 本番AdMob Ad Unit IDに切り替え
  • StoreKit 商品（月額・年額）の本番価格設定
  • RankingService.createRanking() / VoteService.castVote() のEdge Function化検討
  • キャプションテキスト・スクショ画像の最終版確定
  • App Store Connect の Age Rating 確認（投票・通報あり）
  • プライバシーポリシー・利用規約のURLを app-policies/rankingstadium/ に配置
