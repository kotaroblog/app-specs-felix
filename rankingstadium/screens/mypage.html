マイページ

自分のプロフィール・作成ランキング・選択肢申請の管理

v1.3 画面ID: RS-MYPAGE-01

目的・役割

自分のアカウント情報、作成したランキング、選択肢申請の処理（自分が作成者の場合）、下書き編集を行う中心画面。

要素種類説明
プロフィールヘッダVStack表示名・isPremiumバッジ・points・loginStreak

無料作成枠表示TextfreeCreationRemaining
「動画で広告報酬」ButtonRewarded広告 → PointAction.adReward（実装確認）
作成ランキング一覧List（タブ切替） 「作成した」タブで自分が作成したランキングを表示。

・draft / created / recentlyCancelled の3リストを並列再取得して整合させる

予約取り消しダイアログ仕様

項目内容
形式alert（中央モーダル）
タイトル「予約公開を取り消しますか？」
本文（pt消費ありの場合） 公開を取り消すと、今回消費した◯ptは返却されます。
取り消し後、24時間以内は下書きに復元が可能です。
(マイページ→下書き→下書きに戻す)
本文（pt消費なしの場合） 取り消し後、24時間以内は下書きに復元が可能です。
(マイページ→下書き→下書きに戻す)
ボタン「予約を取り消す」(destructive) / 「やめる」(cancel)
取り消し直後の挙動引っ張り更新なしで即「下書き」バッジ表示に切り替わる

実装メモ

  • MyPageViewModel が currentUser とランキング一覧を保持
  • CreateRankingView は自前 NavigationStack を持たないため、MyPageView の NavigationStack で動作
  • 選択肢申請件数で動線を出し分け
  • 「下書きに戻す」直後の「予約公開」バッジ残留バグを修正。draft / created / recentlyCancelled の3リストを並列再取得して状態整合を担保
  • 「作成した」タブはクライアント側で予約公開 → 公開中 → 終了済みの順にソート（各グループ内は作成日時の新しい順）
  • 予約取り消し後、引っ張り更新なしで即「下書き」バッジ表示に切り替わる
