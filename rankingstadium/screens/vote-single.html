シングル投票画面

1つの選択肢を選んで投票するモード（RankingMode.single）

v1.2 画面ID: RS-VOTE-02

目的・役割

選択肢の中から1つを選んで投票する画面。再投票時はポイント消費（revote1: 3pt, revote2: 7pt）。

UI 構成

要素種類説明
RankingHeaderView（Xスタイル）VStack 画面上部の共通ヘッダ。以下7段構成。
1段目: 作成者アバター + 表示名 + フォローボタン（Xスタイル） + 右上に大きなランクバッジ（44px円、Sは虹グラデ）
2段目: 投票タイプバッジ（シングル/TOP3） + クイズ通過バッジ（hasQuiz=true のみ）
3段目: 大タグ
4段目: 小タグ
5段目: 投票期間 + 残り時間
6段目: タイトル（選択肢直前に再表示）
7段目: 説明文（存在する場合のみ）
選択肢グリッドLazyVGrid各 Choice をカード表示（label + image）
選択ハイライトBorder選択中の Choice を視覚化
投票ボタンButton選択完了でアクティブ化。VoteView のルート VStack 最下部に直接配置（safeAreaInset を完全廃止）
消費ポイント表示Text「投票には Npt を消費します」
バナー広告共通配置MainTabView の tabContentWithBanner(for:) 内で tabContent と VStack(spacing: 0) で並べて配置（プレミアム時は非表示）。旧仕様の safeAreaInset(.bottom) から変更

レイアウト変更（v1.1）

  • VoteView 内の重複タイトル表示は削除（タイトルは RankingHeaderView 6段目に集約）
  • 戻るボタン横の navigationTitle は維持

レイアウト変更（v1.2）

  • 選択肢タップのヒットエリア: 行（HStack + Spacer）全体をヒット可能に（buttonStyle(.plain) + Spacer の SwiftUI 既知問題対策）
  • 投票ボタンの配置を .safeAreaInset から ScrollView 直下 VStack 最下部への直接配置に変更（AdBanner との z-order 逆転解消）

レイアウト変更（v1.3 - TF 1.0(22) 反映）

  • safeAreaInset の二重積み重ね問題を根本対処: MainTabView と VoteView の両方で .safeAreaInset(edge: .bottom) を使うと、iOS 26 + NavigationStack destination 環境で合計領域が画面外にはみ出すバグが発生していた
  • MainTabView 側: safeAreaInset を完全廃止し、新規 tabContentWithBanner(for:) 関数で各タブコンテンツと AdBannerContainer を VStack(spacing: 0) で並べる構造に変更
  • VoteView 側: safeAreaInset を完全廃止し、ルート VStack(spacing: 0) { ScrollView { 選択肢 }; VStack { Divider + 投票ボタン } } 構造に変更
  • 表示順序: [ScrollView 選択肢] > [投票ボタン] > [AdBanner] > [TabBar]

状態

状態表示内容
未選択投票ボタン無効
選択済投票ボタン有効
投票中ProgressView、ボタン無効
ポイント不足「ポイントが足りません」アラート
既投票検出「すでに投票済みです」 → ResultView へ

遷移

操作遷移先
選択 + 投票ボタン押下VoteService.castVote() → ResultView
キャンセルRankingDetailView へ戻る

使用するデータモデル

Vote {
    let rankingId: UUID
    let userId: UUID
    let choiceId: UUID
    let points: Int       // 0 (初投票) / 3 (revote1) / 7 (revote2)
    let voteNumber: Int   // 1, 2, 3
    let ipHash: String?
    let deviceHash: String?
}

実装メモ

  • VoteService.castVote() の4ステップ（ポイント消費 → INSERT → choice/ranking voteCount RPC）
  • Unique制約違反は VoteError.alreadyVoted として処理
  • 非アトミック処理 → Edge Function 化検討（v1.1）
