検索機能

過去732件の豆知識をキーワード・カテゴリで検索・フィルタリング

v1.0 機能ID: DT-FEAT-SEARCH

概要

v1.0.5 で追加された機能。daily_tips.json に格納された 732件（v1: 366 + v2: 366）の豆知識を、 キーワード（タイトル・カテゴリ）と複数フィルター（お気に入り・既読/未読・カテゴリ別）で検索できる。 検索結果からロックされた豆知識を開いた場合、広告視聴で解放可能。

関連画面

  • ホーム画面（トップバーに🔍ボタン）
  • 検索画面

仕様

項目内容
アクセスホーム画面のトップバー右側「🔍」アイコン
検索対象タイトル + カテゴリ（大文字小文字を区別しない）
フィルターお気に入りのみ・既読/未読・カテゴリ別（横スクロールチップ）
表示形式件数表示 + リスト（カテゴリバッジ＋日付＋タイトル）
日付表記「2025年度 1月1日」「2026年度 1月1日」で年度を区別
ロック対応ロック対象を開いた場合、専用シートで「広告を見て解放」ボタン表示

API

// TipLoaderService
func getAllTips() -> [DailyTip]
func searchTips(keyword: String) -> [DailyTip]
    // タイトル・カテゴリで部分一致（lowercased で大文字小文字区別なし）

処理フロー

  1. ホーム画面の🔍ボタンタップ → SearchView がシートで表示
  2. onAppear で cachedFavorites(Set<String>) と allCategories を初期化（パフォーマンス対策）
  3. キーワード入力 → results computed property で TipLoaderService.searchTips() を呼ぶ
  4. フィルターチップで絞り込み（カテゴリ・お気に入り・既読/未読）
  5. 結果リストから tip をタップ → SearchResultDetailSheet で詳細表示
  6. ロック対象の場合は「広告を見て解放する」ボタンを表示
  7. 動画視聴完了 → HistoryService.markAsAdViewed() → 通常の TipCardView に切り替わる

パフォーマンス最適化

body 評価のたびに FavoriteService.getFavorites() や TipLoaderService.getAllTips() を呼ぶと キーワード入力時に重くなるため、@State でキャッシュ化（onAppear で1回ロード）。

ビジネスルール

  • 検索結果は v1/v2 両方表示（「2025年度」「2026年度」で区別）
  • 結果から開く際、過去日付のロックは尊重（カレンダー画面と同じ判定）
  • ロック解放はリワード広告視聴 + HistoryService への記録
  • プレミアム会員は全件閲覧可
