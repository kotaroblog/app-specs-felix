学習メモ機能

各日付に300字以内のメモを保存できる学習日記機能

v1.0 機能ID: DT-FEAT-MEMO

概要

各日付に対して300字以内の「学習メモ」を保存できる。豆知識を読んだ気づき・覚えたいこと・関連する自分の体験などを記録する学習日記として機能。 カレンダー画面に📝マークで可視化され、過去の振り返りやリテンション向上に寄与する。

関連画面

  • ホーム画面（カレンダーセルに📝マーク）
  • カレンダー画面（同上）
  • 豆知識カード（メモボタン）
  • メモ編集画面（シート表示・MemoEditView）

仕様

項目内容
文字数上限300字（リアルタイムカウンター表示・超過時は保存ボタン無効化）
編集可能範囲過去・当日いずれも自由（最古日付 2025/4/1 以降）
保存先AppGroup UserDefaults
カレンダー表示メモあり日付の右上に 📝 マーク
削除編集画面の「メモを削除」ボタン（確認ダイアログ付き）

API

@MainActor
final class MemoService {
    static let shared: MemoService
    static let maxLength = 300

    func getMemo(for date: Date) -> String?
    func hasMemo(for date: Date) -> Bool
    func getAllMemoDateKeys() -> Set<String>
    func setMemo(_ text: String, for date: Date)
    func removeMemo(for date: Date)
}

データ構造（UserDefaults）

キー型用途
memo_text_yyyy-MM-ddString該当日付のメモ本文（最大300字）
memo_dates[String]メモ登録済み日付リスト（インデックス用）

処理フロー

  1. ユーザーが豆知識カードの「メモ」ボタンをタップ
  2. MemoEditView がシートで表示される
  3. onAppear で MemoService.shared.getMemo(for:) から既存メモを読み込み
  4. ユーザーがTextEditorで入力（300字以内・リアルタイムカウンター）
  5. 「保存」タップで setMemo 実行 → AppViewModel.reloadMemoDates()
  6. カレンダー再描画時、メモあり日付に📝マークが表示される

ビジネスルール

  • 空文字を保存すると removeMemo と同等扱い
  • 300字超過時は自動的に prefix で切り詰め
  • カレンダーの範囲外日付（2025/4/1 以前）にはメモを保存しない（UI でタップ不可）
  • メモは AppGroup 経由でWidgetからも参照可能だが、現状Widget表示には未使用

容量見積もり

1メモ最大300字（UTF-8で約900バイト）× 365日 × 数年 でも数百KB程度。UserDefaults の容量制限内（数MB）に十分収まる。

関連型・連携

連携先用途
AppGroup UserDefaultsメモ本文と日付インデックスの永続化
AppViewModel.memoDateKeysカレンダー再描画用のキャッシュ（Set<String>）
